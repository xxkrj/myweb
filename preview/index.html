이지민

제작 동기

이 프로젝트는 평소에 패킷 트레이서 등 기존에 사용되는 프로그램들에서 표현되는 정적 라우팅은 직관적으로 이해하기 어렵게 표현되어 있다고 생각해서, 이러한 단점을 보완하기 위해서 기획하게 되었습니다.

프로젝트 영상(전체화면 권장)


프로젝트 설명

이 프로그램은 크게 4가지 과정으로 진행됩니다. 첫 번째 과정은 토폴로지를 구성하는 라우터의 수를 지정하는 것입니다.

위 코드를 통해서 최소 4개, 최대 7개의 라우터를 설정합니다.
라우터의 수는 방향키를 통해서 입력받고, 그 값을 비교해서 개수를 설정합니다.
두 번째 과정은 각 라우터에 네트워크 주소를 지정하는 것입니다.

위 코드를 통해서 지정한 라우터 수에 따른 토폴로지의 모양을 출력합니다.
setpixel함수를 통해서 라우터의 모양을 자세하게 표현해주었습니다.

토폴로지 모양이 출력된 후, 위 코드를 통해서 각 라우터에 네트워크 주소를 차례대로 지정합니다.
그 다음, 세 번째 과정은 라우팅 테이블을 작성하는 것입니다.

라우터의 라우팅 테이블을 확인하거나 핑을 전송할 수 있습니다.
방향키를 통해 다른 라우터를 선택할 수 있습니다.

위 코드는 라우팅 테이블 보기를 선택했을 때 실행되는 부분입니다.
해당 라우터의 라우팅 테이블이 표시됩니다.

편집하기를 선택했을 때 실행되는 코드입니다.
다른 라우터의 주소와 그 라우터로 가기 위한 포트를 차례대로 입력받습니다.
이 과정을 반복하여 모든 라우팅 테이블 설정을 완료합니다.
마지막 과정은 핑을 전송하는 것입니다.

위 코드는 핑을 보내기 위한 설정을 하는 부분입니다.
출발지 주소와 도착지 주소를 입력하고, for문을 통해서 출발지를 알아냅니다.

출발지 주소를 알아낸 뒤, 위 코드를 통해 앞에서 작성한 라우팅 테이블에 따라 핑을 전송합니다.
핑이 전송되는 과정은 토폴로지에 초록색으로 표시됩니다.

코드 세부 설명

라우터의 모든 정보가 들어가는 구조체 파트입니다.
router 구조체는 해당 라우터의 주소, 라우팅 테이블, 그리고 포트로 이루어져 있습니다.
라우팅 테이블에는 다른 라우터의 주소와, 그 라우터로 가기 위한 포트가 저장됩니다.
포트에는 포트 번호와 그 포트와 연결된 라우터의 번호가 저장됩니다.

포트의 정보가 저장되는 부분입니다.
지정한 라우터의 수에 따라서 라우터 구조체의 포트 부분에 다른 정보가 할당됩니다.

프로그램에 사용된 간단한 함수들입니다.
gotoxy함수는 커서의 위치를 지정해주는 함수입니다.
setcolor함수는 출력되는 내용의 색을 바꿔주는 함수입니다.
cussor,cussor2함수는 각각 커서를 보이거나 안보이게 해주는 함수입니다.

위의 두 함수는 각각 토폴로지의 라우터 색을 변경하는 함수입니다.
전체 라우터의 수와, 몇 번 라우터인지를 구분하여 색을 변경하여 다시 출력합니다.

개인용 VPN

박동진

프로젝트 개요

항상 사용하는데 위험이 있거나 자잘한 버그들이 계속 생겨 사용에 차질이 많이 생기는 무료 VPN말고
나만의 게임에서도 사용할 수 있고 일반 웹에서도 사용할 수 있는 VPN 만들기

세부 설명

VPN 또는 가상 사설 네트워크는 인터넷을 통해 디바이스 간에 사설 네트워크 연결을 생성합니다.
VPN은 퍼블릭 네트워크를 통해 데이터를 안전하게 익명으로 전송하는 데 사용됩니다.
또한 사용자 IP 주소를 마스킹하고 데이터를 암호화하여 수신 권한이 없는 사람이 읽을 수 없도록 합니다.

신원 보호 및 개인의 프라이빗 보호, 안전한 퍼블릭 인터넷을 위해 등등
여러 가지 이유로 VPN을 만들 수 있지만 이번 경우에는 한국에서 다른 나라의 서버를 접속하기 위하여 만들었습니다.

프로젝트 영상(전체화면 권장)


리눅스 설정

#!/bin/bash

sysctl_content="""vm.swappiness = 0
net.ipv4.neigh.default.gc_stale_time=120
net.ipv4.conf.all.rp_filter=0
net.ipv4.conf.default.rp_filter=0
net.ipv4.conf.default.arp_announce = 2
net.ipv4.conf.all.arp_announce=2
net.ipv4.tcp_max_tw_buckets = 5000
net.ipv4.tcp_syncookies = 1
net.ipv4.tcp_max_syn_backlog = 1024
net.ipv4.tcp_synack_retries = 2
net.ipv4.conf.lo.arp_announce=2
net.ipv4.ip_forward = 1
net.ipv4.conf.default.accept_redirects = 0
net.ipv4.conf.default.send_redirects = 0
net.ipv4.conf.default.accept_source_route = 0"""

l2tp_psk_content="""conn L2TP-PSK-NAT
rightsubnet=vhost:%%priv
also=L2TP-PSK-noNAT
conn L2TP-PSK-noNAT
authby=secret
pfs=no
auto=add
keyingtries=3
dpddelay=30
dpdtimeout=120
dpdaction=clear
rekey=no
ikelifetime=8h
keylife=1h
type=transport
left=$(hostname -I)
leftprotoport=17/1701
right=%%any
rightprotoport=17/%%any"""

xl2tpd_content="""[global]
listen-addr = $(hostname -I)
ipsec saref = yes
[lns default]
ip range = 192.168.100.128-192.168.100.254
local ip = 192.168.100.99
require chap = yes
refuse pap = yes
require authentication = yes
name = LinuxVPNserver
ppp debug = yes
pppoptfile = /etc/ppp/options.xl2tpd
length bit = yes"""

xl2tpd_option_content="""ipcp-accept-local
ipcp-accept-remote
ms-dns 8.8.8.8
ms-dns 8.8.4.4
noccp
noauth
crtscts
idle 1800
mtu 1410
mru 1410
nodefaultroute
debug
lock
proxyarp
connect-delay 5000"""

# Install xl2tpd
yum install epel-release -y
yum install xl2tpd libreswan -y

printf "$sysctl_content" >> /etc/sysctl.conf
sysctl -p

printf "$l2tp_psk_content" >> /etc/ipsec.d/l2tp_psk.conf

read -p "Please Input pre-shared key(PSK): " shared_key
echo "$(hostname -I) %any: PSK \"$shared_key\"" >> /etc/ipsec.secrets

# Turn of IPSec
ipsec setup start
ipsec verify
systemctl enable ipsec

# Configure xl2tpd
mv /etc/xl2tpd/xl2tpd.conf /etc/xl2tpd/xl2tpd.conf.old
mv /etc/ppp/options.xl2tpd /etc/ppp/options.xl2tpd.old
printf "$xl2tpd_content" > /etc/xl2tpd/xl2tpd.conf
printf "$xl2tpd_option_content" > /etc/ppp/options.xl2tpd

read -p "Please Input Username: " username
read -p "Please Input Password: " password
echo -e "$username * $password *\n" >> /etc/ppp/chap-secrets

# Start xl2tpd
systemctl start xl2tpd
systemctl enable xl2tpd
systemctl status xl2tpd

# Set firewall
firewall-cmd --permanent --direct --add-rule ipv4 filter INPUT 0 -p gre -j ACCEPT
firewall-cmd --permanent --zone=public --add-masquerade
firewall-cmd --permanent --add-rich-rule='rule protocol value="esp" accept'
firewall-cmd --permanent --add-rich-rule='rule protocol value="ah" accept'
firewall-cmd --permanent --add-port=1701/udp
firewall-cmd --permanent --add-port=500/udp
firewall-cmd --permanent --add-port=4500/udp
firewall-cmd --permanent --add-service="ipsec"
firewall-cmd --reload

# Set iptable
interface_d=`ip route get 8.8.8.8 | sed -nr 's/.*dev ([^\ ]+).*/\1/p'`
iptables -A INPUT -p gre -j ACCEPT
iptables -A OUTPUT -p gre -j ACCEPT
iptables -A FORWARD -m state --state RELATED,ESTABLISHED -j ACCEPT
iptables -A FORWARD -s 192.168.100.0/24 -j ACCEPT
iptables -t nat -A POSTROUTING -s 192.168.100.0/24 -o $interface_d -j MASQUERADE
iptables-save > /etc/sysconfig/iptables

# Register Startup
echo -e "iptables-restore /etc/sysconfig/iptables\n" >> /etc/rc.d/rc.local
chmod +x /etc/rc.d/rc.local

setsebool -P daemons_use_tty 1

호버크래프트

신동준

프로젝트 개요

양방향 무선통신 SPI를 사용해 전진, 우현, 좌현, 릴레이 모듈을 이용한 LED on/off 등
다양한 동작을 수행할 수 있는 호버크래프트 제작

세부 설명

조종기에서 조이스틱과 포텐쇼미터, 택트 스위치를 통해 값을 입력받고 각각의 값을 통해 전진, 후진, 좌현, 우현, 릴레이모듈 on/off등 실행할 동작을 결정한다. 결정한 값을 String 형태의 메시지로 이어붙여 rf24통신 모듈을 통해 호버크래프트 본체에 전송한다, 본체에서 값을 받으면 파싱을 통해 받은 값을 보낼 때의 형태로 분류한다. 분류한 값을 필요한 형태(int)로 변경해서 정리한다. 정리된 값에 따라 if문을 통해 각 기능을 수행한다.

프로젝트 영상(전체화면 권장)


Part-Hardware

위 사진은 완성된 호버크래프트의 모습이다.
후면부의 로터 2개의 힘으로 이동하게된다.

무선으로 호버크래프트를 조종하는 컨트롤러이다.
좌측 상단 스위치 : 스위치를 통해 호버크래프트 본체에 설치된 릴레이 모듈을 제어 또는 실행할 모드를 설정
좌측 하단 조이스틱 : 호버크래프트 본체를 전진, 좌현, 우현, 오른쪽, 왼쪽으로 회전 동작
우측 하단 가변저항 : 조이스틱을 통한 전진 동작 시 속도 제어

수신된 데이터를 LCD에 표시해 통신 상태를 쉽고 빠르게 확인할 수 있도록 설계했다.

인체 저항 리듬게임 컨트롤러

도민혁

프로젝트 개요

평소 리듬게임을 즐겨하던 저는 더 잘해질 수 있는 방법을 찾다가
오락실처럼 컨트롤러를 이용해 플레이하면 더욱 재밌을 것 같다는 생각이 들었습니다.

또 입력 지연이 기존 키보드보다 낮으면 내려오는 노트를 더욱 빠르게 클릭할 수 있어서
더 잘할 수 있겠다고 생각이 들었습니다.

마지막으로 아두이노 레오나르도를 사용해 푸쉬버튼을 눌러 키보드를 입력하는 것보다
인체 저항을 이용해서 입력되는 방식이 흥미롭고 이 방식이 리듬게임에 최적화라고 생각해서 만들게 되었습니다.

세부 설명

컨트롤러는 정전용량을 감지하는 방식이 아닌, 인제 저항에 흐르는 직류전류를 감지하는 방식입니다.
임계값을 적절히 설정하면 오작동도 거의 일어나지 않습니다.
아날로그 핀이 7개가 있다면 7K도 가능합니다.

동테이프 30M를 사용해 전기를 통하도록 만들었으며 직류전류를 통하게 하기 위해서
자신의 신체 중 일부분에 연결을 해줘야 합니다.(발밑에 금속 판이 제일 좋습니다)

디자인은 어떤식으로 할까 고민하다가 옛날 B급만화감성 느낌이 좋을 것 같아서
더 이상 안보고 팔기도 애매한 만화책을 뜯어서 장식했습니다.

프로젝트 영상(전체화면 권장)


Part-Hardware

먼저 필요한 동테이프를 A4용지에 하나하나 붙여주었습니다.

제작에 필요한 박스를 준비해 주고 흰 A4용지로 덮어 주었습니다.

위와 같이 동테이프를 잘라준 뒤 아날로그 핀에 전류가 연결될 수 있도록 붙여주었습니다.

중간과정입니다.

5V핀에다가는 몸에 연결해 직류전류가 흐를 수 있도록
이렇게 긴 선으로 연결을 해주었습니다.

전체적인 회로도를 완성시킨 뒤에, 위처럼 박스에 구멍을 뚫어서
아날로그 핀이 나올 수 있도록 했습니다.

lcd가 들어갈 구멍도 뚫어주었습니다.

디자인 후 완성된 컨트롤러의 모습입니다.

Part-Software

각각 외부정적변수를 선언해 준 후, 모든 경우의 수에 해당하는
키 조합을 눌러서 나오는 아날로그 최소값과 최대값을 확인하고,
그 두 값을 구분하기 위한 적절한 기준인 threshold 변수 값을 정해주었습니다.
count는 lcd에 표시할 숫자입니다.

시리얼 통신 선언을 해준 뒤 CLKnocked를 false로 설정함으로써
누르지 않는 판정으로 정해주었습니다.

시리얼 모니터 확인을 해줍니다.
if문을 들여다보면 터치를 했을 경우 키보드 Q가 설정이 됩니다.
false이였던 CLKnocked를 true으로 변경하여 누른 판정이 됩니다.
터치 시 lcd에 표시할 카운트 수를 1추가합니다.
터치를 하지 않았을 경우 키보드 Q는 CLKnocked는 false가 되어 누르지 않는 판정이 됩니다.
이 if문을 각각 핀번호와 설정한 값에 따라서 6개를 추가해 주었습니다..

마지막으로, count수를 lcd에 표시해줍니다.
